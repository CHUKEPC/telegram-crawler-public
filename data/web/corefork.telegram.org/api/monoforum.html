Monoforum

Telegram supports direct messages to channels, implemented as follows.

While in the API direct channel message groups are sometimes called "monoforums" they actually share more similarities with the saved messages API.

Direct channel messages can be toggled using channels.updatePaidMessagesPrice, passing the ID of the channel to channel: the same method can also be used to set a price in Telegram Stars users must pay to send direct messages to the channel (or to send messages to the supergroup, see paid messages »).

Once enabled, the channel.linked_monoforum_id will be automatically populated with the ID of a new special supergroup, also called a monoforum.

To send a direct message to the channel, users simply have to send a message to the associated monoforum: unlike for normal supergroups/forums there's no need to join it.

Monoforums have the following special characteristics:

  • The channel.id of a monoforum will have a range different from those of normal supergroups and channels (this is especially relevant when generating and categorizing the associated bot API ID, see here » for the correct procedure to use for monoforum IDs).
  • The channel.monoforum flag will be set for a monoforum.
  • The channel.linked_monoforum_id of a monoforum will be set to the ID of the associated channel.

All messages sent to the monoforum are split into "topics", reusing the same UI used for tabbed forums (note: monoforums should use the tabbed forum UI regardless of the value of channel.forum_tabs).

Each topic is associated to exactly one user.

  • Administrators of the channel can view and reply to all messages sent by all topics of the monoforum.
  • Non-administrators can only view and send messages to their own topic.

Monoforum topics do not share their implementation with forum topics (i.e. they are not based on message threads): instead, topics are identified by the saved_peer_id field of messages and messageServices, which will be equal to the ID of the user.

To reply to incoming monoforum messages, when invoking messages.sendMessage and the other methods, channel administrators must pass an inputReplyToMonoForum to reply_to, with monoforum_peer_id equal to the topic ID (aka saved_peer_id): the resulting message will have a saved_peer_id equal to the passed monoforum_peer_id (equal and set for both for the administrator, and for the receiving user).
To reply to a message within a monoforum topic, pass inputReplyToMessage with the reply_to_msg_id and other flags set as usual, and monoforum_peer_id equal to the topic ID (aka saved_peer_id); unlike for normal forums, do not populate top_msg_id.
