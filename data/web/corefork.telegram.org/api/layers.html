Layer 181

This layer introduces fact-checks », Telegram Stars and collapsible message quotes ».

Schema changes

New Methods
  • Added messages.editFactCheck - Edit/create a fact-check on a message.
  • Added messages.deleteFactCheck - Delete a fact-check from a message.
  • Added messages.getFactCheck - Fetch one or more factchecks, see here » for the full flow.
  • Added payments.getStarsTopupOptions - Obtain a list of Telegram Stars topup options » as starsTopupOption constructors.
  • Added payments.getStarsStatus - Get the current Telegram Stars balance of the current account (with peer=inputPeerSelf), or the stars balance of the bot specified in peer.
  • Added payments.getStarsTransactions - Fetch Telegram Stars transactions.
  • Added payments.sendStarsForm - Make a payment using Telegram Stars, see here » for more info.
  • Added payments.refundStarsCharge - Refund a Telegram Stars transaction, see here » for more info.
New Constructors
  • Added factCheck - Represents a fact-check » created by an independent fact-checker.
  • Added starsTransactionPeerUnsupported - Describes a Telegram Star transaction that cannot be described using the current layer.
  • Added starsTransactionPeerAppStore - Describes a Telegram Star transaction with the App Store, used when purchasing Telegram Stars through the App Store.
  • Added starsTransactionPeerPlayMarket - Describes a Telegram Star transaction with the Play Store, used when purchasing Telegram Stars through the Play Store.
  • Added starsTransactionPeerPremiumBot - Describes a Telegram Star transaction made using @PremiumBot (i.e. using the inputInvoiceStars flow described here »).
  • Added starsTransactionPeerFragment - Describes a Telegram Star transaction with Fragment, used when purchasing Telegram Stars through Fragment.
  • Added starsTransactionPeer - Describes a Telegram Star transaction with another peer (usually a bot or a channel).
  • Added inputStorePaymentStars - Used to top up the Telegram Stars balance using the Play Store/App Store flow (official apps only).
  • Added starsTopupOption - Telegram Stars topup option.
  • Added inputInvoiceStars - Used to top up the current account's Telegram Stars balance.
  • Added starsTransaction - Represents a Telegram Stars transaction ».
  • Added payments.starsStatus - Info about the current Telegram Star balance and transaction history ».
  • Added updateStarsBalance - The current account's Telegram Stars balance » has changed.
  • Added payments.paymentFormStars - Represents a payment form, for payments to be using Telegram Stars, see here » for more info.
  • Added payments.paymentReceiptStars - Receipt for payment made using Telegram Stars.
Changed Constructors
  • Added factcheck parameter in message
  • Changed type of provider from string to flags.3?string in inputMediaInvoice
  • Added flags, collapsed parameters in messageEntityBlockquote

Layer 177

This layer adds the following features:

  • Channel ad revenue »: Telegram channel owners can now receive 50% of the revenue from ads displayed in their channels, as well as view detailed revenue statistics.
  • Telegram Business connected bots »: connected business bot action bar » and bot-side constructors.
  • Telegram Business introduction »
  • Telegram Business chat links »
  • Fetch info about Fragment collectibles »
  • Birthdays »
  • Linked personal channels »
  • Report ads »
  • Disable ads on boosted channels you own »
  • Replace single stickers in stickersets »
  • Fetch all stickersets owned by the current account »
  • A refactored and improved channel/supergroup/group invitation flow »
  • Improvements for keyboardButtonRequestPeer.
    Now bots will also receive the selected peer's name, username, photo (if present and requested by the new bot-side inputKeyboardButtonRequestPeer button).
    Previously, just the peer's ID was sent, which could only be used to fetch the name/photo if the bot had already previously cached the peer's access hash ».
    • Bots will now receive the requested info as a messageActionRequestedPeerSentMe constructor instead of the old messageActionRequestedPeer constructor, which will only be received by users that shared the info.
    • Bots must now use inputKeyboardButtonRequestPeer to send keyboardButtonRequestPeer buttons.
